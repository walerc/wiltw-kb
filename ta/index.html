🕯️ K线形态检测

算法原理

K线形态的检测是完全基于规则的布尔条件判断。每根K线有四价（开/高/低/收），形态就是这四价的几何关系。

核心计算：实体与影线

实体长度 = |收盘 - 开盘| 上影线长度 = 最高价 - max(开盘, 收盘) 下影线长度 = min(开盘, 收盘) - 最低价 当日振幅 = 最高价 - 最低价

强度定义

每个信号附带0-100的强度评分，基于形态偏离理想形态的程度：

  • 影线类：影线/实体比越大 → 强度越高（下影4x实体 = 80分）
  • 吞没类：D2实体远超D1 → 强度越高
  • 多K线类：实体递增幅度越大 → 强度越高
📐 图表形态检测

核心思路：拐点检测 + 几何约束

图表形态的检测分为两个独立步骤，每个步骤参数透明可调。

第1步：拐点检测

局部峰值 = 前后各N天最高价的最大值 = 该点 局部谷值 = 前后各N天最低价的最小值 = 该点 N = 5 (日线，过滤日间噪声)

为防过近拐点干扰，间距<8天的同类拐点只保留更极端的那个，并强制峰值-谷值交替排列。

第2步：序列匹配

提取到的拐点序列与已知形态的几何约束逐一匹配：

形态方向拐点序列几何约束
双顶看跌 峰→谷→峰 两峰<3%偏差 | 谷在峰下3-30% | 跌破谷
双底看涨 谷→峰→谷 两谷<3%偏差 | 峰在谷上3-30% | 突破峰
头肩顶看跌 峰→谷→峰(头)→谷→峰(肩) 中间峰最高 | 两肩<5%偏差 | 颈线合理 | 跌破右谷
倒头肩底看涨 谷→峰→谷(头)→峰→谷(肩) 中间谷最深 | 两肩<5%偏差 | 颈线合理 | 突破右峰
对称三角形中性 峰↘谷↗ ≥3递降峰 + ≥3递升谷 → 收敛
上升三角形看涨 峰≡谷↗ ≥3递升谷 + 峰值水平(偏差<5%)
下降三角形看跌 峰↘谷≡ ≥3递降峰 + 谷值水平(偏差<5%)

强度定义

基于形态幅度：双顶回撤越大→强度越高；头肩顶头部深度越大→强度越高。三角形基于收敛质量和拐点数量。

📏 支撑/阻力检测

三线并查，交叉验证

系统同时运行三种独立算法，避免单方法片面性。

方法1：水平支撑/阻力

1. 扫描最近200天的所有局部极值（高点→阻力候选，低点→支撑候选） 2. 相近价格聚类（<1%偏差视为同一价位） 3. 触及≥3次的价位 → 有效水平位 4. 当前位置下方→支撑，上方→阻力

方法2：趋势线支撑/阻力

1. 找连续的3个递升谷值 → 连线 → 上升趋势线(支撑) 2. 找连续的3个递降峰值 → 连线 → 下降趋势线(阻力) 3. 用线性回归拟合趋势线斜率 4. 当前价格在线上方=支撑，下方=阻力

方法3：均线支撑/阻力

使用MA20/MA60/MA120/MA250： - 价格在MA上方 → MA是支撑 - 价格在MA下方 → MA是阻力 - 统计60天内价格穿越该MA的次数 → 触及越多=越有效 - 长周期MA(≥120)默认加分（心理锚定效应）

综合判断

三种方法的支撑/阻力线汇总后，按当前价格距离排序。最近的支撑和最近的阻力构成"价格通道"。三条以上重叠的价位视为强支撑/强阻力。
